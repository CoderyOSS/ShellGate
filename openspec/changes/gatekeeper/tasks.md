## Layer 3: Core Services

### Main branch

- [ ] 4.1 Implement command classifier: parse command name + args into structured action (gh:pr:create, git:push, git:local, api:read, api:write, unknown)

- [ ] 4.2 Implement repo extractor: parse `--repo` flag from gh/git args, glob matching for repo patterns

- [ ] 4.3 Implement grant CRUD: create, read, list, revoke grants with TTL, max_uses, repo pattern matching

- [ ] 4.4 Define wire protocol: JSON length-prefixed frames for request (command, args, cwd, pid) and response (action, env, approval_id, reason)

- [ ] 4.5 Implement request handler: classify command → permission check → respond with allow/allow+env/pending/reject

- [ ] 4.6 Implement pending response flow: store pending channel per approval_id, block on channel, resolve when user approves/rejects

- [ ] 4.7 Connection lifecycle: clean up on disconnect, handle multiple concurrent clients

- [ ] 4.8 Implement host-side command executor: spawn `gh.real` with args and GH_TOKEN, capture stdout/stderr/exit_code

- [ ] 4.9 Implement output streaming: stream stdout/stderr chunks back over the unix socket to the patched zsh

- [ ] 4.10 Implement socket keepalive: send ping frames every 60s during long-running commands

- [ ] 4.11 Handle command timeout: configurable max execution time (default 30 min), kill process on timeout

### zsh branch

- [ ] 4.12 Create zsh patch: add gate hook in `Src/exec.c` before fork for external commands, calling a gate_check function that connects to `/run/gate.sock`

- [ ] 4.13 Implement gate_check in new `Src/gate.c`: serialize command + args to JSON, send to unix socket, parse response

- [ ] 4.14 Implement allow path: if response is allow, proceed with normal fork+execve

- [ ] 4.15 Implement allow+env path: if response includes env vars, set them in child process after fork, before execve

- [ ] 4.16 Implement pending path: poll socket until resolved, print "gate: waiting for approval..." to stderr

- [ ] 4.17 Implement reject path: print reason to stderr, skip execution, set exit code to 1

- [ ] 4.18 Implement local git fast-path: detect git subcommands in {add,commit,status,diff,log,branch,stash,checkout,merge,rebase,reset,tag} and skip socket round-trip

- [ ] 4.19 Implement proxy mode for gh: when command is `gh`, send full command to gate-server, stream output back

- [ ] 4.20 Test patched zsh: verify allow, allow+env, pending, reject, local git, gh proxy paths

### REST API

- [ ] 4.21 Implement REST API with axum on port 3000: `/api/approvals`, `/api/grants`, `/api/audit`, `/api/permissions`, `/api/config`

- [ ] 4.22 Implement approval endpoints: GET list (with status filter), POST approve, POST reject

- [ ] 4.23 Implement grant endpoints: POST create, GET list, DELETE revoke

- [ ] 4.24 Implement audit log endpoint: GET with pagination, action filter, text search, date range

- [ ] 4.25 Implement permission config endpoints: GET defaults, PUT update default, POST reset-to-defaults

- [ ] 4.26 Implement SSE endpoint for real-time approval updates (`/api/events`)
